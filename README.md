# Study Tools Bot

Телеграм-бот для ведения школьных предметов, оценок и прогноза среднего балла, с лёгкой админ-панелью на FastAPI.

## Основной функционал

- Регистрация пользователя через `/start` с автосозданием базовых предметов.
- Управление предметами через `/subjects`:
  - просмотр списка (с тематическими эмодзи),
  - добавление предмета (с возможностью отмены),
  - переименование (с кнопкой копирования старого названия),
  - удаление.
- Работа с оценками через `/grades`:
  - средний балл по каждому предмету,
  - общий средний по активному периоду,
  - встроенный счётчик для быстрого добавления/удаления оценок (с отображением дельты и удобными колонками),
  - действия: сохранить, отменить изменения (сброс) и отменить последнее сохранение (Undo).
- Калькулятор оценок через `/calc`:
  - статистика по каждому предмету,
  - общий средний,
  - рекомендуемая итоговая оценка (школьное округление),
  - прогноз, сколько оценок нужно до целевого уровня.
- Настройки периодов через `/settings`:
  - выбор системы периодов (`trimesters` или `quarters`),
  - ручной выбор активного периода или возврат в авто-режим.
- Веб-админка:
  - вход по логину/паролю,
  - список пользователей,
  - время регистрации и последней активности,
  - блокировка и разблокировка пользователей.
- Контроль доступа:
  - заблокированным пользователям закрыт доступ в middleware и в `/start`.

## Технологии

- Python 3.12
- aiogram 3
- SQLAlchemy (async) + asyncpg
- Alembic (миграции)
- FastAPI + Jinja2 (админка)
- Docker Compose
- pytest + pytest-asyncio

## Структура проекта

- `bot/` - хендлеры, middleware и утилиты бота.
- `admin/` - FastAPI-приложение админки, auth, роуты и шаблоны.
- `database/` - SQLAlchemy-модели и базовая конфигурация БД.
- `alembic/` - конфигурация и ревизии миграций.
- `tests/` - асинхронный тестовый набор (бот + админка).
- `config/` - настройки приложения из `.env`.

## Быстрый старт

### 1) Настройка окружения

Скопируйте `.env.example` в `.env` и заполните обязательные переменные:

- `BOT_TOKEN`
- `POSTGRES_*`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_SECRET_KEY`

### 2) Запуск через Docker

```bash
docker compose up --build -d
```

Сервисы:

- сервис `bot`, контейнер `study_tools_bot_app`
- сервис `postgres`, контейнер `study_tools_bot_postgres`
- сервис `admin`, контейнер `study_tools_bot_admin`, админка: `http://127.0.0.1:8080/admin/login`

### 3) Применение миграций

```bash
docker compose exec -T bot alembic upgrade head
```

### 4) Запуск тестов

```bash
docker compose exec -T bot python -m pytest tests/ -v
```

## Команды бота

- `/start` - регистрация пользователя и приветствие.
- `/subjects` - управление предметами.
- `/grades` - сводка оценок и работа со счётчиком.
- `/calc` - калькулятор среднего и прогноза.
- `/settings` - система и активный период.
- `/help` - справка.

## Админ-панель

- Страница входа: `/admin/login`
- Основная страница: `/admin/users`
- Доступные действия:
  - заблокировать пользователя,
  - разблокировать пользователя,
  - выйти из админки.

Учётные данные берутся из `.env` (`ADMIN_USERNAME`, `ADMIN_PASSWORD`).

## Примечания

- При старте бот также вызывает `Base.metadata.create_all()` для удобства локальной разработки.
- В рабочих окружениях источником истины по схеме БД остаются миграции Alembic.
